# Thread 4: Funding the People Who Keep Ethereum Alive

1/12
Some public goods are projects.

Others are people doing sustained, high-context work over years.

Ethereum core protocol work is a clear example: clients, research, specs,
testing, coordination, upgrades, and security reviews all depend on human
continuity.

2/12
Protocol Guild is one of the most important experiments in funding that layer.

Instead of funding one deliverable, it supports Ethereum core contributors as a
group, using a public registry and long-term token vesting.

3/12
This matters because core protocol work is not always grant-shaped.

Maintainers answer reviews, debug edge cases, attend calls, refine specs, mentor
contributors, and reduce risk long before any single feature gets a launch
announcement.

4/12
Traditional grants often struggle with this.

They like milestones.
But maintenance is continuous.

The work is valuable because it compounds quietly: fewer outages, better
coordination, stronger clients, safer upgrades, and faster response when things
go wrong.

5/12
Protocol Guild's model recognizes that the commons is made of contributors.

Funding goes to individuals doing Ethereum L1 research, development, and
coordination work, rather than only to project wrappers.

That is a different unit of allocation.

6/12
The public registry is important.

It makes the set of supported contributors visible, inspectable, and easier to
reason about.

Public goods funding needs legitimacy. Transparency helps donors understand what
they are actually supporting.

7/12
Long-term vesting is also a design choice.

It says: protocol work should not depend only on short campaigns or one-off
grant cycles.

Maintainers need durable support because the systems they maintain are durable
dependencies.

8/12
This pattern can spread beyond Ethereum.

Every ecosystem has core maintainers:

- client engineers
- security responders
- spec authors
- release coordinators
- infra operators
- docs maintainers
- governance process stewards

Find them before they burn out.

9/12
The risk is that "fund the insiders" can become opaque or exclusionary.

The antidote is clear criteria, open contribution records, reviewable membership
processes, and routes for new contributors to become visible over time.

Guilds need permeability.

10/12
Protocol Guild points toward a bigger idea:

Public goods funding should match the shape of the work.

Bounties fit discrete tasks.
Grants fit planned projects.
Retro funding fits proven impact.
Guild funding fits ongoing stewardship.

11/12
Crypto needs all four.

If we only fund shiny new work, the base layer decays.
If we only fund incumbents, new contributors stall.
If we only fund retroactively, early risk becomes harder.

Healthy ecosystems build funding portfolios.

12/12
CTA:

Make a list of the maintainers your project depends on.

Not the logos. The humans.

Then ask how your protocol, DAO, or company can route durable support to the
people keeping your dependencies alive.
